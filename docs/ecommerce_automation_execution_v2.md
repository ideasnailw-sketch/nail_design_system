# 全类目电商自动化闭环系统 2.0（可落地执行版）

> 目标：把“选品-开发-上架-运营-复盘”从概念升级为**可直接搭建的软件/表格/本地自动化后台**。

## 0. 系统总览

```text
数据层 -> 判断层 -> 生成层 -> 执行层 -> 复盘层
```

- **数据层**：采集外部平台 + 店铺 + 客服 + 库存数据。
- **判断层**：统一打分与标签（新品/趋势/爆品/淘汰）。
- **生成层**：AI 自动生成标题、卖点、SKU、FAQ、上架资料。
- **执行层**：自动触发上架、调价、改图、补货、清仓、下架。
- **复盘层**：将销售结果反哺下一轮产品开发。

---

## 1. 最小可运行版本（MVP）

第一版只做 6 张表（CSV/Excel/SQLite 三选一）：

1. `product_master`（产品主表）
2. `trend_watch`（趋势监控表）
3. `hot_score`（爆品评分表）
4. `listing_materials`（上架资料表）
5. `sales_review`（销售复盘表）
6. `inventory_alert`（库存预警表）

### MVP 每日流程（人工 + AI 半自动）

1. 每天收集 10 个趋势品写入 `trend_watch`
2. 跑“趋势评分脚本”产出 top3 候选
3. 用模板自动生成 SKU 方案与上架文案
4. 上架后 7 天拉回销售数据
5. 进入复盘：爆品扩展、平款优化、差品淘汰

## 2. 数据层（Data Layer）落地设计

## 2.1 数据源映射

| source | 关键字段 | 用途 |

## 3. 判断层（Rule Engine）

## 3.1 标签规则（可配置）

- **新品**：上架 ≤ 14 天，且收藏/加购有增长
- **趋势品**：多平台出现 + 搜索/提及增长率超过阈值
- **爆品**：销量高 + 转化高 + 退款可控
- **平款**：有销量但低于目标线
- **滞销**：高库存 + 低销量 + 周转天数超阈值
- **淘汰品**：低点击 + 低转化 + 差评集中

## 3.2 爆品评分公式（建议）

```text
hot_score =
0.30 * 销量得分 +
0.25 * 转化得分 +
0.15 * 增长得分 +
0.10 * 利润得分 +
0.10 * 复购得分 -
0.10 * 风险扣分
```

评分分层：
- S：85+
- A：75-84
- B：65-74
- C：50-64
- D：<50

## 4. 生成层（AI Content Engine）

## 4.1 自动生成对象

- 商品方向、系列结构、SKU 裂变矩阵
- 标题、五点卖点、详情页模块文案
- 主图提示词、短视频脚本、客服 FAQ
- 上架批量导入表、复盘建议

## 4.2 上架内容模板

### 标题公式

```text
核心关键词 + 颜色/材质 + 功能/场景 + 人群 + 产品词
```

### 五点卖点模板

1. 风格与外观
2. 适用场景
3. 材质与品质
4. 规格与套装内容
5. 礼赠/售后/便捷性

### 详情页结构

1. 第一屏：主图 + 一句话卖点
2. 第二屏：适合谁
3. 第三屏：使用场景
4. 第四屏：细节与工艺
5. 第五屏：尺寸规格
6. 第六屏：套装内容
7. 第七屏：使用方法
8. 第八屏：FAQ 与评价
9. 第九屏：品牌承诺

## 5. 执行层（Automation Workflow）

## 5.1 自动动作清单

- 自动上架（批量写入平台模板）
- 自动改标题（按关键词点击表现）
- 自动换主图（按 CTR 低阈值）
- 自动调价（按转化率/利润率）
- 自动补货与清仓提醒
- 自动下架淘汰品

## 5.2 生命周期触发器

- 新品 7 天无点击 -> 换主图
- 新品 14 天无加购 -> 改标题/价格
- 21 天无订单 -> 进入淘汰候选
- 连续 14 天高转化 -> 升级爆品池
- 爆品可售天数 < 7 -> 补货预警
- 60 天库存不动 -> 清仓策略
- 90 天无改善 -> 强制下架

## 6. 复盘层（Learning Loop）

每周自动生成 `weekly_review`：

- Top 爆品共性（颜色/尺寸/场景/价格带）
- 滞销与退款主因（质量/尺寸/预期不符）
- 客诉关键词变化（推动详情页与 FAQ 更新）
- 下周开发清单（新增/优化/淘汰）

### 复盘反哺规则示例

- 裸色销量占比 > 40% -> 下轮增加裸色系
- 礼盒转化高于散装 20% -> 增加礼盒 SKU
- 尺寸问题工单占比 > 15% -> 强化尺寸可视化

---

## 7. SKU 裂变引擎

通用公式：

```text
1个母版 = 颜色 × 尺寸 × 材质 × 场景 × 包装 × 套装 × 价格带
```

示例（36 SKU）：

```text
3色 × 2尺寸 × 2包装 × 3套装 = 36 SKU
```

新增约束：
- 自动去重（同质 SKU 不重复）
- 自动利润过滤（毛利率低于阈值不生成）
- 自动风险标记（侵权/物流/季节性）

---

## 8. 库存与淘汰自动化

### 安全库存公式

```text
安全库存 = 最近7天日均销量 × 补货周期天数 × 1.3
```

### 淘汰策略流水线

1. 改主图
2. 改标题
3. 改价格
4. 组合销售
5. 清仓促销
6. 下架
7. 记录失败原因库

失败原因库用于下轮选品避坑（关键词/价格带/质量/包装/渠道匹配）。

## 9. 软件化实施路径（30/60/90 天）

## Day 1-30（MVP）
- 建 6 张核心表
- 跑每日趋势采集 + 基础评分
- 完成半自动上架模板输出

## Day 31-60（自动化）
- 引入任务调度（定时采集、定时评分）
- 接入客服问题分类与库存预警
- 增加生命周期自动触发器

## Day 61-90（闭环）
- 接入平台 API（或批量导入导出）
- 建立复盘报告自动生成
- 形成“开发-上架-复盘-再开发”稳定循环

## 10. 本地部署建议（与你当前仓库兼容）

可沿用当前仓库模式：**CSV + Python + 本地 HTML**。

建议新增脚本：
- `scripts/sync_trends.py`：趋势采集与写入 `trend_watch.csv`
- `scripts/rule_engine.py`：标签与评分计算
- `scripts/generate_listing_assets.py`：批量生成上架资料
- `scripts/run_lifecycle_jobs.py`：生命周期自动动作
- `scripts/build_review_report.py`：周复盘报告

建议新增目录：
- `data/raw/`（原始抓取）
- `data/processed/`（清洗后）
- `outputs/reports/`（周报/月报）
- `configs/rules.yaml`（阈值与规则可配置）

## 11. 系统最终定义

> 全类目电商自动化产品系统 = 数据发现机会 + AI 生成商品 + 平台测试销售 + 客户反馈复盘 + 库存控制 + 爆品放大 + 淘汰品清理 + 下一轮新品迭代。

每天系统自动回答 8 个问题：

1. 今天什么在涨？
2. 哪些新品刚冒头？
3. 哪些趋势值得跟？
4. 哪些产品已进入爆发？
5. 哪些商品应立即优化？
6. 哪些库存存在风险？
7. 哪些产品应淘汰？
8. 下一轮该开发什么？
